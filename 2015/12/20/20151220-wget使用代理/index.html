wget使用代理

发表于 2015-12-20 | 分类于 Linux

方法一：使用环境变量

$ export http_proxy=http://127.0.0.1:8118

方法二：使用配置文件

为wget使用代理，可以直接修改/etc/wgetrc，也可以编辑~/.wgetrc。

#You can set the default proxies for Wget to use for http, https, and ftp.
# They will override the value in the environment.
https_proxy = http://127.0.0.1:8118/
http_proxy = http://127.0.0.1:8118/
ftp_proxy = http://127.0.0.1:8118/

# If you do not want to use proxy at all, set this to off.
use_proxy = on

这里 use_proxy = on 开启了代理，如果不想使用代理，每次都修改此文件未免麻烦，我们可以在命令中使用-Y参数来临时设置：

-Y, --proxy=on/off           打开或关闭代理

方法三：使用-e参数

wget本身没有专门设置代理的命令行参数，但是有一个”-e”参数，可以在命令行上指定一个原本出现在”.wgetrc”中的设置。于是可以变相在命令行上指定代理：

-e, --execute=COMMAND   执行`.wgetrc'格式的命令

例如：

wget -c -r -np -k -L -p -e "http_proxy=http://127.0.0.1:8118" http://www.foolengineer.com

这种方式对于使用一个临时代理尤为方便。
